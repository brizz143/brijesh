Introduction
1.1 Purpose
The purpose of this document is to specify the requirements for the Bus Reservation System (BRS) which will allow users to book, modify, and cancel bus tickets, view schedules, and make payments online. The document aims to provide both functional and non-functional requirements that will guide the development and implementation of the system.


Scope
The Bus Reservation System will cater to the needs of passengers, bus operators, and administrators by providing an interface for online reservations. It will include features such as seat booking, fare calculation, ticket cancellation, and real-time bus schedule tracking.

Functional Requirements
3.1 User Registration and Login
3.1.1: The system shall allow users to create an account using their email and phone number.
3.1.2: Users shall be able to log in using their registered credentials.
3.1.3: The system shall allow users to reset their password if they forget it.
